# カケイボキ 基本設計書（仕様書）

## 1. システム概要
「カケイボキ」は、複式簿記（ダブルエントリー）の原則に基づいて個人の収支および資産状況を管理するWebアプリケーションです。
Single Page Application (SPA) として構築されており、直感的なUI（グラスモーフィズム・ダークテーマ）と、Firebaseによるリアルタイムクラウド同期を備えています。

## 2. システムアーキテクチャ
*   **フロントエンド**: HTML5, CSS3, Vanilla JavaScript (フレームワーク非依存)
*   **バックエンド / データベース**: Firebase Authentication, Cloud Firestore
*   **PWA (Progressive Web App)**: Service Worker (`sw.js`), Web Manifest (`manifest.json`) を用いたインストール機能およびオフラインでの利用対応
*   **データストア方式**: ローカルストレージ（`localStorage`）とFirestoreを併用したハイブリッド型。オフライン時も動作し、オンライン復帰時・ログイン時にデータを同期します。

## 3. ディレクトリ・ファイル構成
| ファイル名 | 役割 |
| :--- | :--- |
| `index.html` | アプリケーションのメイン画面・UI構造 (SPAとして全画面を内包) |
| `style.css` | アプリケーション全体のスタイル・テーマ定義 |
| `app.js` | コアロジック（データ管理、UI描画、入力制御、イベントハンドリングなど） |
| `firebase-config.js` | Firebase SDK の初期化設定 |
| `sw.js` | PWA用 Service Worker (キャッシュ制御など) |
| `manifest.json` | PWA設定ファイル（アイコン、テーマカラー、表示モードなど） |

## 4. データ構造 (Cloud Firestore / LocalStorage)
データはユーザーごとに一元管理されます。

*   **ユーザーコレクション**: `users/{userId}`
*   **ドキュメント構造**:
    *   `accounts` (Array): 勘定科目リスト
        *   `id` (String): ID (英数字)
        *   `name` (String): 科目名
        *   `type` (String): `asset`(資産), `liability`(負債), `equity`(純資産), `income`(収益), `expense`(費用)
    *   `journals` (Array): 仕訳データリスト
        *   `id` (String): 一意のID (タイムスタンプベース等)
        *   `date` (String): 取引日付 (YYYY-MM-DD)
        *   `description` (String): 摘要 (内容)
        *   `debits` (Array): 借方リスト `[{ accountId, amount }]`
        *   `credits` (Array): 貸方リスト `[{ accountId, amount }]`
        *   `recurringId` (String): (任意) 定期仕訳から生成された場合のテンプレートID
    *   `recurring` (Array): 定期仕訳テンプレートリスト
        *   `id` (String): テンプレートID
        *   `description` (String): 定期仕訳の摘要
        *   `dayOfMonth` (Number): 毎月の発生日 (1-28など)
        *   `debits` (Array): 借方リスト
        *   `credits` (Array): 貸方リスト
        *   `enabled` (Boolean): 有効/無効フラグ
    *   `lastUpdated`: 最終更新日時 (Firestore Server Timestamp)

## 5. UI / 画面構成 (タブ切替式SPA)
*   **仕訳入力**: 複数行の借方/貸方入力、リアルタイム残高チェック
*   **仕訳帳**: 期間指定による仕訳一覧
*   **総勘定元帳**: 選択した勘定科目ごとの取引履歴および残高推移
*   **貸借対照表 (B/S)**: 基準日における資産・負債・純資産の集計と当期純利益の算出
*   **損益計算書 (P/L)**: 指定期間における収益・費用の集計と純損益
*   **定期仕訳**: 毎月の自動記帳テンプレート設定
*   **勘定科目**: 勘定科目マスタの管理 (追加・変更・削除)

## 6. クラス設計・コアロジック
*   `DataStore` クラス:
    `app.js` 内で定義され、ローカルストレージの読み書き、Firestoreとの`onSnapshot`によるリアルタイム同期、各データのCRUD操作（追加・更新・削除・残高計算）を司ります。
*   残高計算:
    資産・費用の科目は借方（Debit）をプラス、負債・純資産・収益の科目は貸方（Credit）をプラスとして集計ロジックが組まれています。
